How to Test Utils

Option 1 — Django Shell

Open the Django shell and paste any utility directly from the book to test it immediately.

python manage.py shell
Option 2 — Utils Files

Paste utilities into the matching file in utils/. Each file maps to a category (queries, models, managers, views, etc.). Then import and use them in your views, models, or shell.

Full Shell Example

Here is a complete example using get_or_none from the Query Utilities chapter. Follow the same pattern for any utility in the book.

Step 1 — Paste the utility into the matching file

File: utils/query.py

from django.db.models import Model
from typing import Type, TypeVar
 
T = TypeVar("T", bound=Model)
 
def get_or_none(model: Type[T], **kwargs) -> T | None:
    try:
        return model.objects.get(**kwargs)
    except model.DoesNotExist:
        return None
Step 2 — Open the Django shell
python manage.py shell
Step 3 — Import and test
# import the utility you just pasted
from shop.utils.query import get_or_none
 
# import a model from the starter project
from shop.models import Product
 
# test it — product exists (seed_data created 20 products)
product = get_or_none(Product, id=1)
print(product)  # returns Product object
 
# test it — product does not exist
missing = get_or_none(Product, id=99999)
print(missing)  # returns None instead of raising DoesNotExist

Follow the same three steps for every utility in the book. Paste into the matching utils file, open the shell, import and test.
